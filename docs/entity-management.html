エンティティ管理

Person（個人）・Corporation（企業）・Organization（組織）の3種類のエンティティを登録・編集・削除する方法を解説します。

Person（個人）の管理

1

WordPress管理画面のサイドバーから「Author Panel Manager」を開き、Personタブをクリックします。

2

新規追加ボタンをクリックして、Person登録画面を開きます。

3

各フィールドを入力し、保存をクリックして登録を完了します。

Personのフィールド一覧

フィールド名 説明
名前 表示名として使用される日本語の氏名です。
英語名 構造化データ（JSON-LD）に出力される英語表記の名前です。
Role Author（著者）、Reviewer（レビュアー）、Editor（編集者）、Contributor（寄稿者）、Creator（作成者）、Translator（翻訳者）から選択します。
肩書き 役職や専門分野などの肩書きを入力します。
自己紹介 著者パネルに表示される自己紹介文です。
画像 プロフィール画像をメディアライブラリから選択またはアップロードします。
URL 個人サイトやプロフィールページのURLです。
sameAs URL SNSや外部プロフィール（X、LinkedIn等）のURLを複数登録できます。構造化データのsameAsプロパティに出力されます。
パネルデザイン 著者パネルの表示デザインを選択します。
Roleの選び方

記事執筆者はAuthor、レビュー担当はReviewer、編集者はEditorを選択してください。適切なRoleを設定することで、構造化データが正しく出力され、検索エンジンに正確な情報を伝えられます。

Corporation（企業）の管理

Corporationタブでは、企業情報を登録・管理できます。Publisher（発行者）やSponsor（スポンサー）など、記事に関連する企業エンティティを設定します。

Corporationのフィールド一覧

フィールド名 説明
名前 企業の日本語名称です。
英語名 構造化データに出力される英語表記の企業名です。
Role Publisher（発行者）、Sponsor（スポンサー）など、記事における企業の役割を選択します。
説明 企業の概要や事業内容を記述します。
URL 企業の公式サイトURLです。
ロゴ 企業ロゴ画像をメディアライブラリから選択またはアップロードします。
sameAs URL 企業の公式SNSアカウントや外部プロフィールのURLを複数登録できます。
パネルデザイン 著者パネルの表示デザインを選択します。

Organization（組織）の管理

Organizationタブでは、研究機関、NPO、業界団体などの組織情報を管理します。フィールド構成はCorporationと同一ですが、企業以外の組織を登録する際に使用します。

使用中の記事の確認

エンティティ編集画面の下部には「使用中の記事」セクションが表示されます。そのエンティティがどの記事で参照されているかを一覧で確認できます。

削除前の確認

エンティティを削除する前に、必ず「使用中の記事」セクションを確認し、参照しているすべての記事からエンティティの紐付けを解除してください。

アンインストール時のデータ削除

プラグインをアンインストールすると、3つのカスタムテーブル（wp_apm_persons、wp_apm_corporations、wp_apm_organizations）がすべて削除されます。必要なデータは事前にバックアップしてください。
