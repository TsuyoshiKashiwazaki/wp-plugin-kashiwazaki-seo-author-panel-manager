トラブルシューティング

よくある問題とその解決方法をまとめています。

よくある問題と解決策

著者パネルが表示されない

原因: ショートコードにエンティティIDが未指定、または指定したIDのエンティティが存在しない。

1

ショートコードに指定したIDが正しいか確認してください。

2

管理画面でエンティティが実際に存在するか確認してください。

3

プラグインが有効化されているか確認してください。

JSON-LDが出力されない

原因: Customモードでtarget_schema_idが未指定。

1

mode="custom"を指定している場合は、target_schema_idも必ず指定してください。

2

Standardモードの場合は、ページのソースコードでapplication/ld+jsonを検索し、出力を確認してください。

3

Customモードの場合は、<head>内にJSON-LDが出力されているか確認してください。

Gutenbergブロックでエンティティが表示されない

原因: REST APIの権限が不足している。

1

現在のユーザーにedit_posts権限があるか確認してください。

2

エンティティが管理画面から正しく登録されているか確認してください。

3

ブラウザで/wp-json/kapm/v1/personsにアクセスし、APIレスポンスを確認してください。

パネルのスタイルが崩れる

原因: テーマのCSSとの競合。

1

ブラウザの開発者ツールで.kapm-author-panelのスタイルを確認してください。

2

テーマのCSSで競合しているプロパティを調整してください。

3

panel-style.cssの読み込み順序が正しいか確認してください。

ヒント

デフォルトテーマ（Twenty Twenty-Fourなど）に一時的に切り替えることで、テーマ固有の問題かプラグインの問題かを切り分けられます。

動作要件

項目 要件
WordPress 6.0以上
PHP 8.0以上
対応ブラウザ Chrome / Firefox / Safari / Edge 最新版
管理権限 manage_options
ブロック権限 edit_posts

技術仕様

データ保存

項目 内容
カスタムテーブル 3テーブル（prefix_apm_persons / prefix_apm_corporations / prefix_apm_organizations）
wp_options 未使用

フロントエンド

項目 内容
CSS panel-style.css（約4KB）
JavaScript なし
JSON-LD（Standard） パネル直後に出力
JSON-LD（Custom） wp_head（priority 99）で出力

REST API

エンドポイント メソッド 権限
/kapm/v1/persons GET edit_posts
/kapm/v1/corporations GET edit_posts
/kapm/v1/organizations GET edit_posts

セキュリティ

対策 内容
nonce検証 すべての管理画面リクエストでnonce検証を実施
権限チェック manage_options / edit_postsによる権限制御
入力サニタイズ sanitize_text_field等による入力値の無害化
出力エスケープ esc_html / esc_attr / esc_url等による出力エスケープ
